# Design Context — Airflux Agent Platform

## Target Audience
AB180 internal team: developers, PMs, data scientists, and ops engineers who manage AI agents. Technical users comfortable with dashboards and configuration. Korean-speaking, English UI labels acceptable.

## Use Cases
- Register, configure, and monitor AI agents
- Tune prompts and evaluate agent quality
- View costs, latency, error rates at a glance
- Manage scheduled agent runs and review logs

## Brand Personality
**Industrial precision meets warm approachability.** This is a control room, not a marketing site. It should feel like a well-designed mission control: information-dense but not overwhelming, functional but not sterile. Think Bloomberg Terminal's density married with Linear's refinement.

## Aesthetic Direction
- **Tone**: Utilitarian-refined. Dense information, clear hierarchy, zero decoration for decoration's sake.
- **Colors**: Zinc-based neutrals with a single warm accent (amber/orange) for active states and alerts. Dark mode default.
- **Typography**: Geist Sans for UI, Geist Mono for metrics/IDs/timestamps.
- **Density**: Compact. Admins want information, not whitespace.
- **Differentiation**: The sidebar shows live agent status with subtle pulse indicators. The overview page uses a dense grid layout inspired by observability dashboards, not marketing hero sections.
